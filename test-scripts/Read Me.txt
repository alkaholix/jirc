NOTES: I realize you can make a connection/auth to this new server 
without using sockets. The /nick <new nick> command is enabled. Channel 
names are case-sensitive. The server is a BETA version so don't expect
it to be perfect. If their are bugs, it is most likely on their 
end, not yours.

You can use this connection in conjunction with your Buzzen v1 Connection.
Simply load the .mrc file in your existing script and type /server -m and it 
will not interfere with your pre-existing Connection. Then in the new status 
window type /BV2 and voila.

KNOWN ISSUE: When another user quits from one room and stays in another 
room (this is common with webchatters) mIRC interprets it as they quit from 
the server and removes them from every common channel's nicklist. They are 
still in the room but not in the nicklist. I have addressed this with
the buzzen developers and hope it is resolved. In the mean time, just /hop
to refresh nicklist.

To Connect:

Type: /BV2

- Can type Again at anytime to reconnect


To Store Login Info:

Type: /BV2.login 

- Can also use popups


To Join a Bot:

Type: /BV2.bot <channel name> <bot username>

- Can join multiple bots to one room.
- Bot sockname format: BOT[username]*


To Join Multiple Rooms at Once:

Type: /join %#Room1,%#Room2,%#Room3,%#Etc

--------------------------------------

To change TOPIC,ONJOIN,OWNERKEY, etc.

In the channel you wish to modify:

Type: //prop # LOCKED

- This will show you what /prop commands are locked.

To Unlock:

Type: //prop # UNLOCK <property>

- <property> being OWNERKEY, TOPIC, ONJOIN, or w/e.

To modify these properties once unlocked.

Type: //prop # <property> <value>

- Example: //prop # TOPIC We Chat Alot! ... //prop # ONJOIN Welcome to the Chat Room!

--------------------------------------

Channel (Room) Creation:

I have it setup to auto create a channel if it doesnt exist. But
if you must know, all you have to type is /create %#Channel to
create a channel. You can not specify any other parameters than 
channel name. (At least to my current knowledge.)

Changing Category:

By default your channel will be created in GN (General). 

Type (in channel): //prop # UNLOCK CATEGORY
Type (in channel): //prop # CATEGORY <new category abbreviation>

Then change the topic,onjoin message, etc. the way described above.

- Category Abbreviations:

UN - Unlisted (No Category)
GN - General
RM - Romance
CP - Computing
PR - Peers
IN - Interests
NE - News & Events
SP - Sports & Politics
CC - City Chats
LS - Lifestyles
ET - Entertainment
RL - Religion
RP - RolePlay

--------------------------------------

Access Lists:

- Level can be: OWNER,HOST,VOICE,GRANT,DENY

Add: //access # add <level> <mask>

Delete: //access # delete <level> <mask>

Clear: //access # clear <level> 

- Leaving <level> blank clears ALL access.

List: //access # list
